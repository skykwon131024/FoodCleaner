## Day 1 — 문제를 찾아라

### p.1 🗺️ 디자인씽킹 5단계
1. 💛 **공감** — 사용자 입장에서 불편함 찾기
2. 🔵 **정의** — 문제를 명확히 하기
3. 💚 **아이디어** — 해결 방법·기능 떠올리기
4. 🟠 **프로토타입** — 빠르게 화면 그려보기
5. 🔴 **테스트** — 발표, 피드백하기

- Day 1: 공감 단계(⭐) / Day 2: 이후 단계(◆)

### p.2 🧠 컴퓨팅 사고력 4요소
| 요소 | 뜻 |
| --- | --- |
| 🧩 분해 | 큰 문제를 작게 나누기 |
| 🔍 패턴 인식 | 반복되는 규칙 찾기 |
| 🎯 추상화 | 불필요한 것 빼기 |
| 📋 알고리즘 | 순서 정하기 |

- 흐름: 큰 문제 → 분해 → 패턴 → 추상화 → 알고리즘 → 해결!

### p.3 💛 공감 체험 — 🧩 분해하기
**미션: "학교 급식 대기 시간을 줄여라!"** (원인을 잘게 쪼개기)
1. 줄이 하나밖에 없다
2. 오늘 메뉴를 모른다
3. 식판 반납구가 하나뿐이다
4. 메뉴 고르는 시간이 길다
5. 키오스크 부족
6. 배식대 부족 (⭐)
7. 주문 방법이 키오스크밖에 없다 (⭐)

### p.4 🔍 패턴 찾기 — 원인 그룹화
- **그룹 1 (시간 관련)**: 메뉴 고르는 시간이 길다
- **그룹 2 (시설 관련)**: 키오스크 부족, 배식대 부족, 주문 방법이 키오스크밖에 없다
- **그룹 3 (정보 관련)**: 메뉴를 모른다

### p.5 🎯 추상화 + 📋 알고리즘
- **핵심 문제(추상화)**: 시설을 확장할 필요가 있다
- **해결 절차(알고리즘)**: 모금을 받는다 → 키오스크를 산다 → 키오스크를 설치한다

### p.6 발표 메모 + 깨달음
- 다른 조 발표를 들으며 인상 깊은 아이디어·배울 점 메모
- 공감 체험 활동을 통한 오늘의 깨달음 기록

### p.7 📱 앱 해부하기 — 역방향 분석
- **분석 앱**: 유튜브 / **조원**: 김은찬, 권정범, 김도건
1. **해결하는 문제/사용자**: 심심한 사람들, 영상을 공유하고 싶은 사람들
2. **핵심 기능 3가지**: 업로드, 쇼츠, 시청
3. **메인 화면 구성**: 알고리즘을 통해 사용자에게 여러 영상을 띄움
4. **개선할 점**: 광고가 많다
5. **최근 업데이트**: 앰비언트 모드 & 화면 밝기(모바일 영상 어두워짐 현상 개선)

### p.8 🔵 문제 정의서 (1/2) — 단계별 질문
1. **누구의 문제?** 나의 문제
2. **언제/어디서?** 방학 때 뭘 해야 하는지 메모에 적을 때
3. **어떤 문제?** 실천을 안 한다
4. **왜?(원인)** 귀찮아서
- **합치기**: ❶은(는) ❷에서 ❸를 겪고 있다. 왜냐하면 ❹이기 때문이다.

### p.9 🔵 문제 정의서 (2/2) — 5 Whys
- **문제**: 방학 때 하려는 것이 있는데 실천을 안 한다
1. 왜? → 귀찮아서
2. 왜? → 힘들어서
3. 왜? → 계속 까먹어서
4. 왜? → 할 시간이 안 나서 (⭐)
- **🔑 근본 원인**: 귀찮아서

### p.10 ✊ 마무리 — 과제 + 개발자 마인드셋
**과제(4/18까지)**
1. 문제 정의서 보완 → 페들렛 수정본 업로드
2. 나머지 문제 2개도 정의서 작성
3. 3개 중 최종 1개 확정(Day 2에 사용)
4. 다른 친구 문제에 ❤️ + 댓글

**개발자 마인드셋 5가지**
1. 완벽보다 시작
2. 실패는 데이터
3. 사용자 먼저
4. 혼자보다 함께
5. 윤리적 책임

## Day 2 — 기획하고 만들어라 (4월 18일 토, "나는 이 앱의 기획자")

### p.11 📝 Day 1 복습 퀴즈
- Q1. 디자인씽킹 5단계: 공감, 정의, 아이디어, 프로토타입, 테스트
- Q2. 컴퓨팅 사고력 4요소: 분해, 패턴 인식, 추상화, 알고리즘
- Q3. 좋은 문제 정의 프레임워크 문장
- Q4. 5 Whys란?

### p.13 💚 사용자 페르소나
- **이름·나이**: 홍길동 — 23세
- **현재 불편함**: 냉장고에 사둔 식재료를 활용할 줄 모른다
- **바라는 점**: 있는 식재료를 활용한 요리를 알려주면 좋겠다
- **성격·습관**: 상상력이 풍부하지 않다 (⭐)
- **디지털 사용 습관**: 검색을 귀찮아 한다 (⭐)
- **관심사**: 요리 (⭐)
- **한마디**: "가진 식재료를 입력하면 만들 수 있는 음식을 알려주면 좋겠다"

### p.14 💚 사용자 여정 맵 (⚡ = 불편 포인트)
1. 식재료를 산다 — 기쁨
2. 냉장고에 넣는다 — 행복
3. 뭘 만들지 모른다 — 불안 ⚡
4. 식재료가 썩어 간다 — 귀찮음 ⚡
5. 검색하기 귀찮다 — 슬픔 ⚡
6. 결국 버렸다 — 슬픔
- ⚡ 표시 지점 = 앱이 개입할 수 있는 지점

### p.15 💚 핵심 기능 도출
1. 사둔 식재료 사진을 찍으면 그 식재료를 활용한 요리를 알려준다
2. 알려준 요리의 요리 방법을 알려준다
3. 산 식재료의 유통기한을 알려준다
4. 원하는 요리 스타일(일식/양식/한식)을 입력하면 그런 요리를 알려준다 (⭐)

### p.16 💚 ⭐ 기능 우선순위 매트릭스
- ① 반드시 구현(중요·쉬움) → 최우선 개발
- ② 도전(중요·어려움) → 멘토와 함께
- ③ 여유 있으면(안 중요·쉬움)
- ④ 과감히 빼기(안 중요·어려움)

### p.17~18 🟠 화면 스케치 (1~4)
- 핵심 화면 3장 필수 / 5장 도전(⭐)
- 구성 요소: 📦 헤더, 🔘 버튼, 📝 입력창, 📋 리스트, 🖼️ 이미지, 🔽 네비

### p.19 🟠 화면 흐름도 (네비게이션)
- 화면 간 이동 경로를 화살표로 연결
- 3~4번 터치 안에 핵심 기능 도달이 이상적

### p.20 📋 ⭐ 프로젝트 기획서 (1/2)
- **① 프로젝트명**: 푸드 클리너
- **② 문제 정의**: 냉장고에 그냥 있다가 버리게 되는 식재료가 많다
- **③ 핵심 기능 3개**
  1. 사둔 식재료 사진을 찍으면 활용 요리를 알려준다
  2. 핸드폰 앱으로 식품 정보 확인 가능
  3. 산 식재료의 유통기한을 알려준다
- **④ 기대 효과**: 버리게 되는 식재료가 없어진다

### p.21 📋 ⭐ 프로젝트 기획서 (2/2)
- **⑤ 핵심 화면 스케치** (p.17~18 참고)
- **⑥ 대상 사용자**: 검색을 귀찮아 하고 사둔 식재료를 활용할 줄 모르는 사람 (⭐)
- **⑦ 예상 일정**: 5월 / 6~8월 / 9~11월 배포 (⭐)
- 이 기획서가 5월 바이브코딩 → 6~11월 멘토링의 출발점

### p.22 🎤 🔴 피칭 준비 시트
**3문장 피칭(필수)**
1. "나는 [문제]를 해결하는 [앱 이름]을 기획했다."
2. "핵심 기능은 [1], [2], [3]이다."
3. "이 앱이 있으면 [기대 효과]."

**5단계 정식 피칭(⭐)**: 인사 → 문제 → 해결 → 효과 → 마무리
